# Perbaikan Tampilan "Data Servis" v2.0.2

## Masalah
Kartu "Data Servis" sekarang terlihat seperti form admin lama:
- Judul plain "Data Servis" tanpa ikon/pembatas
- 3 input field bertumpuk dengan label hint panjang ("Nama/Tipe Perangkat (misal: Samsung A51)")
- Radio button 3 baris vertikal, tiap baris penuh teks panjang → makan tempat & berantakan
- Tombol HITUNG & Nota Baru sejajar, Nota Baru kurang jelas fungsinya
- "Simpan sebagai Template" tombol text kecil terpisah

## Solusi — Desain modern ala app service professional

### 1. Header kartu dengan ikon + garis pembatas
- Ganti "Data Servis" plain jadi: ikon 📱 + label "Data Servis" + subtitle kecil "Isi detail perangkat & layanan"
- Tambah garis tipis (`View` height 1dp, bg slate_divider) di bawah header untuk memisahkan

### 2. Input field lebih ringkas
- Hint dipersingkat: "Perangkat (mis. Samsung A51)", "Nama Pelanggan", "Modal Sparepart"
- Tambah `app:startIconDrawable` di tiap field (ikon kecil di kiri): perangkat → ic_template (sementara), pelanggan → ic_people, sparepart → bisa pakai prefix Rp yang sudah ada
- Jarak antar field naik dari 8dp → 12dp

### 3. Level Jasa → Chip/Card selectable 3-kolom (BUKAN radio button)
Ganti RadioGroup vertikal 3 baris jadi **3 kartu kecil berdampingan horizontal** (grid 3 kolom), tiap kartu:
- Label besar "L1" / "L2" / "L3" di atas
- Label kecil "Ringan" / "Menengah" / "Berat" di bawah
- Selected state: background blue_primary + teks putih
- Unselected: background white/blue_light + teks slate

Ini paling professional dan hemat tempat vertikal. **Tantangan**: perlu ubah cara bacanya di Kotlin (cek kartu mana yang selected, bukan `rgServiceLevel.checkedRadioButtonId`). Saya simpan ID `rbLevel1/2/3` di kartu (sebagai tag atau view id) supaya `selectedLevel()` minimal berubah, atau buat helper baru yang baca tag/selected state.

### 4. Tombol aksi dirapikan
- **HITUNG**: tombol primary penuh lebar, ikon kalkulator, tinggi lebih tinggi (52dp), teks 16sp bold
- **Nota Baru**: jadi tombol sekunder di bawah HITUNG (bukan sejajar), outline, teks "Simpan & Baru"
- **Simpan Template**: tetap text button, tapi pakai ikon disket di kiri

### 5. Detail teknis level jasa
Karena kartu L1/L2/L3 tidak muat deskripsi panjang ("flashing, bypass" dll), deskripsi dipindah jadi helper text kecil di bawah grid: "L1: Ringan • L2: Menengah • L3: Berat" — atau hilangkan saja (sudah ada di label kartu).

## File yang diubah
1. `app/src/main/res/layout/activity_main.xml` — redesain block "INPUT TEKNISI"
2. `app/src/main/java/com/ketantech/kalkulatorservis/MainActivity.kt`:
   - Ganti `selectedLevel()` baca dari kartu yang dipilih (pakai tag atau track selected view)
   - `setupInputs()`: ganti listener RadioGroup → click listener tiap kartu
   - `onSaveInstanceState`/`restoreState`: simpan/pulihkan level terpilih via state kartu
   - Hapus binding `rgServiceLevel` (atau tetap simpan RadioGroup tersembunyi? Lebih bersih dihapus)
3. `app/src/main/res/values/strings.xml` — singkat hint, tambah label chip level
4. `app/src/main/res/drawable/` — drawable baru untuk selected/unselected state kartu level (atau pakai `cardBackgroundColor` di kode — lebih simpel pakai state list drawable)

## Pendekatan level card (minimal perubahan Kotlin)
Pakai trik: tetap simpan `RadioGroup` + 3 `RadioButton` **tapi `android:visibility="gone"`** (tersembunyi, untuk state). Kartu-kartu level hanya UI; saat kartu diklik → set `rbLevelX.isChecked = true` + update visual kartu. `selectedLevel()` tidak berubah. Ini paling aman dan minim bug.
